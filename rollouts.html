Action space

The agent operates cryoSPARC entirely by writing its own cryosparc-tools Python — it is given no pre-built job tools and no job catalog. Two tools: run_python executes code in a persistent namespace on the cryoSPARC box (returns stdout, the last expression, and any traceback; a connected cs client and non-leaking inspection helpers are pre-bound so the agent can render and see the observations it needs for interactive decisions), and finish is called once the pipeline reaches a final sharpened map. The tools the agent should use are made clear — the cryoSPARC API and the public tutorial — but predefined domain functions are not provided; the agent works out how to drive the tools itself.

An earlier constrained-tool surface — pre-built create_job / run_job / inspect_* wrappers — is not part of the eval; it was a first-iteration scaffold, superseded by the raw surface above. Across the core T20S tasks the action surface is held fixed; the controlled variable is how much of the tutorial information the agent is given — handheld (tutorial + distilled hints) → tutorial (the tutorial verbatim) → realistic (a PI-style goal with the step-by-step recipe removed).

Task overview

Every task in the benchmark, with what it is and why it is included — a catalog only; per-run inputs, outputs, and grader scores are in the rollout cards below and on the benchmark page. Core tasks run on the T20S tutorial subset (EMPIAR-10025, 20 movies), where a deposited map (EMD-6287) and a by-hand golden trajectory both exist, so a run is fully gradeable. Generalization-preview tasks run other / larger structures with a public deposit but no golden trajectory, so they are graded on the deposit-anchored and reference-free subscores only.
